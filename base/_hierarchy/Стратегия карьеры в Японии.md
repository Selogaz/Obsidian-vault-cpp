# Стратегия: Карьера в Японии (Java + WebDev + ML)

> **Цель:** Tech-роль в японской компании к 2034
> **Текущий уровень:** Middle Java Developer
> **Стек:** Java/Spring Boot, PostgreSQL, Docker, Frontend (React/Vue), ML Deployment

---

## Анализ: Японский Tech-рынок

### Context: Японские компании vs Global

- **Японские ( традиционные):** NTT, Rakuten, Line, Sony — требуют Japanese N2+, legacy стек (Java EE, COBOL), медленный найм
- **Gaishi companies (foreign-affiliated):** Google Japan, Amazon Japan, Microsoft — English-friendly, глобальные практики
- **Startup/VC-backed:** Freee, Money Forward, SmartHR — быстрый рост, modern stack, English OK
- **Remote-first (global):** работа на японские компании из-за рубежа

### Observation: Разрыв между твоим стеком и японским

- Японские enterprise до сих пор любят: Java EE, Struts, Oracle
- Spring Boot — относительно modern для Японии (многие на Spring, но не Boot)
- ML Deployment — редкость, niche skill

---

## Claim: Java/Spring Boot — лучшая точка входа

### Evidence: Почему Java, а не Python/Go

| Критерий | Java | Python | Go |
|----------|------|--------|-----|
| Японские вакансии | 3000+ | 1500+ | 300+ |
| Enterprise adoption | Высокая | Средняя | Низкая |
| Твой уровень | Middle | Начальный | Нет |
| Микросервисы в Японии | Spring Boot | FastAPI (редко) | Go (specфичные) |

### Evidence: Spring Boot в Японии

- Rakuten использует Spring (internal tools)
- Money Forward использует Spring Boot
- Amazon Japan Tech = AWS ecosystem = Java
- LINE Engineering = Java + Kotlin

### Context: Что нужно добавить к Java

- Kotlin (легко после Java, популярен в Японии)
- AWS/GCP сертификация (японские компании мигрируют в cloud)
- Японский tech blog presence

---

## Claim: ML Deployment — стратегическая ниша

### Context: Японский ML рынок

- Японские компании遅 (медленно) внедряют ML
- MLOps / ML Platform Engineering — редкая роль
- твоя дипломная работа = релевантный опыт
- Проблема: нужно 5+ лет опыта, чтобы быть "senior ML engineer"

### Observation: Путь через ML Deployment

1. Java Developer → Senior Java Developer (Japan market)
2. ML Platform/MLOps (свое время) → Японские ML роли
3. Или: ML Platform → Remote ML roles → Japan relocation

### Synthesis: Два пути

**Путь A: Java-focused (реалистичный)**
- 2026-2033: Senior Java Developer (российские/зарубежные компании)
- 2033: Job search в Японии как Senior Java Dev
- Преимущество: больше вакансий, быстрее результат

**Путь B: ML-specialized (амбициозный)**
- 2026-2030: Java + ML Platform (thesis work)
- 2030-2033: ML Engineer (data-intensive company)
- 2033-2034: Японские ML roles (MLOps, ML Platform)
- Преимущество: уникальный skill, меньше конкуренция

## Ключевые наблюдения

1. **Японский стек отстаёт от global на 3-5 лет** — твой современный опыт (Spring Boot, Docker, K8s) уже advanced для Японии
2. **Japanese N2 важнее чем 10 лет опыта** — без языка не возьмут даже senior
3. **English-friendly ≠ No Japanese** — English OK для коммуникации, но Japanese нужен для интеграции в команду
4. **Legacy ≠ Bad** — Japanese companies ценят стабильность. Не пытайся "улучшить" их процессы сразу.
5. **Relocation timeline** — обычно 3-6 месяцев от first interview до relocation
