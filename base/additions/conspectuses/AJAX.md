```js
let load = function(url, callback) {
	let ajax = new XMLHttpRequest();
	ajax.open('GET', url);//настраиваем запрос
	ajax.onload = function() {//начнет выполняться, когда придут данные от сервера
		console.log(this);//this указывает на ajax
		callback(this.responseText);//callback - функция, которая будет вызвана после получения файла
	};
	ajax.send();//отправка запроса к указанному url
};

load('text.txt', function(data) {//вызов функции
	console.log(data);
});
```

### 🔍 Пошаговое объяснение:

#### ✅ **Строка 1**:
Объявляется **функция `load`**, которая принимает два параметра:
- `url` — путь к файлу (например, `'text.txt'`)
- `callback` — функция, которая будет вызвана после получения данных

> На этом этапе **ничего не выполняется** — просто создаётся функция.

---

#### ✅ **Строка 2**:
Создаётся новый экземпляр объекта `XMLHttpRequest` и сохраняется в переменную `ajax`.

> Это «заготовка» для будущего HTTP-запроса.

---

#### ✅ **Строка 3**:
Метод `.open()` **настраивает** запрос:
- метод: `'GET'`
- адрес: значение переменной `url` (например, `'text.txt'`)

> ⚠️ **Важно**: `.open()` **не отправляет** запрос! Он только **подготавливает** его.

---

#### ✅ **Строки 4–7**:
Устанавливается обработчик события `onload` — это функция, которая **выполнится позже**, **когда придут данные** от сервера.

- **Строка 5**: выводит в консоль сам объект `XMLHttpRequest` (`this` указывает на `ajax`)
- **Строка 6**: вызывает переданную `callback`-функцию, передавая ей `responseText` (содержимое файла)

> Эта функция **ещё не выполняется** — она просто «назначена» и будет вызвана **асинхронно**, когда ответ придёт.

---

#### ✅ **Строка 8**:
**Вот здесь происходит отправка запроса!**
Метод `.send()` **фактически запускает HTTP-запрос** к указанному `url`.

> С этого момента браузер уходит "в фон", ждёт ответ от сервера, а основной код продолжает выполняться дальше.

---

#### ✅ **Строка 11**:
**Вызов функции `load`** с двумя аргументами:
- `'text.txt'` → станет значением `url`
- `function(data) { console.log(data); }` → станет значением `callback`

> Это **точка входа** — с этого момента начинается выполнение всего процесса.

---

### 🔄 Порядок выполнения (по времени):

1. **Выполняется строка 11**: вызывается `load('text.txt', ...)`
2. Внутри `load`:
	 - Создаётся `XMLHttpRequest` (строка 2)
	 - Настраивается запрос через `.open()` (строка 3)
	 - Устанавливается обработчик `onload` (строки 4–7) — **но он ещё не вызывается!**
	 - **Отправляется запрос** через `.send()` (строка 8) → браузер начинает загрузку `text.txt`
3. **Функция `load` завершается** — управление возвращается в основной поток.
4. **Параллельно**: браузер ожидает ответ от сервера.
5. **Когда файл загружен**:
	 - Вызывается функция из `onload` (строки 4–7)
	 - Выводится объект `XMLHttpRequest` (`console.log(this)`)
	 - Вызывается `callback(this.responseText)` → то есть `function(data) { console.log(data); }`
	 - В консоль выводится содержимое файла

---

### 💡 Ключевые моменты:

- **`.send()` — это момент отправки запроса.**
- Всё, что внутри `onload`, выполняется **позже**, **асинхронно**.
- Основной код (после `load(...)`) **не ждёт** ответа — он продолжает работать сразу.
