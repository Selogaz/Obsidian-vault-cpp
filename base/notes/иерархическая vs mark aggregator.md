**Иерархическая vs mark aggregator**
—
иерархическая:
- Тема требует вложенной структуры с несколькими уровнями дочерних агрегаторов
- Нужна навигация по подтемам внутри
- Используется для синтеза знаний по широкой области

обычная с mark/aggregator:
- одна ключевая концепция, которая агрегирует связанные заметки
- простая структура без глубокого вложения
- промежуточный узел для связи нескольких атомных заметок
