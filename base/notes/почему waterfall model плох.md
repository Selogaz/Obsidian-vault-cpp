**Почему waterfall model плох**
—
**Плохо работает** для программного обеспечения потому что *невозможно визуализировать* дизайн большой программной системы достаточно хорошо *до её реализации* - изначальный дизайн будет иметь множество проблем. Проблемы **не становится очевидными** до тех пор, пока не начнется реализация.

При [[waterfall model|таком подходе]], разработчикам приходится *обходить проблемы, не меняя дизайн* и это приводит к **огромному росту сложности**.
