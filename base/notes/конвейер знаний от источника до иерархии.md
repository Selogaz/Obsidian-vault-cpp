# Конвейер знаний: от источника до иерархии

Полный lifecycle знания в этом хранилище: источник → конспект → атомарная заметка → иерархия.

```
Источник → Conspectus → Note → Hierarchy
    ↘                            ↗
        Project (применение)
```

## 1. Источник — `sources/`

Контейнер для внешнего материала: книга, статья, видео, курс, подкаст, игра. Содержит метаданные (статус, рейтинг, научность, автор) и служит якорем для всего, что из него извлечено.

Статусы:
- 📥 — inbox
- `🟥` — в планах
- `🟦` — в процессе
- `🟩` — готово

Не обязательно читать целиком. Можно взять 2-3 ключевых идеи, поставить `⚛️`, а затем `🟩`.

```yaml
tags:
  - source/article/resource
  - category/<name>
status: 🟦
rating: 🌔
scientificity: 📢
creator:
  - "[[creator name]]"
```

## 2. Конспект — `base/additions/conspectuses/`

Addition к источнику. Свободное поле для выписок, таймкодов, реакций, схем. Временный слой — из него рождаются атомарные заметки.

```yaml
tags:
  - <source type tag из родителя>
  - mark/log/conspectus
  - <category tags из родителя>
status: 🟦
source:
  - "[[source title]]"
```

## 3. Атомарная заметка — `base/notes/`

**Одна идея — один файл.** Главное правило:

- Self-contained — заметка понятна без контекста
- Ссылка на источник через сноску: `[^1]: [[source title]]`
- Атомарность: не «всё про React», а «условный рендеринг в React»

```yaml
tags:
  - note/basic/primary
  - category/<name>
```

## 4. Иерархия — `base/_hierarchy/`

Агрегатор атомарных заметок по общей теме. Когда на одну тему набирается 3+ заметки, они собираются в hierarchy.

```
🧬 js
  ├── hoisting.md
  ├── closure.md
  └── prototype inheritance.md
```

Hierarchy — единственный тип, где разрешена вложенность (🧬 → 🧬).

## 5. Применение — `projects/`

Заметка может ссылаться на проект, где идея используется. Проект, в свою очередь, рождает новые задачи и вопросы, которые ведут к новым источникам.

## Петля

Система циклическая:

1. Задача → читаешь → создаёшь source note
2. Source → conspectus → атомарные notes
3. Notes → собираются в hierarchy
4. Hierarchy → помогает в проекте
5. Проект → новые задачи → к п.1
