## Рекомендуемая стратегия на 15 минут/день

### Недельный цикл

| День | Активность |
|------|-----------|
| Пн | Новая тема (видео/статья 5 мин) + 1 задача (10 мин) |
| Вт–Чт | Повторение темы + 1-2 задачи |
| Пт | Смешанные задачи (все пройденные темы) |
| Сб | Review недели (перебор задач без решения) |
| Вс | Отдых или свободная практика |

### Цикл по темам (≈10 недель)

1. **Array & String** (2 недели)
2. **Linked List + Stack + Queue** (1.5 недели)
3. **Trees + Heaps** (2 недели)
4. **Graphs** (2 недели)
5. **DP + Backtracking** (2 недели)
6. **Sort + Search + Bit** (1.5 недели)

### Инструменты

#### Ресурсы для практики

| Ресурс | Что бесплатно | Для чего |
|--------|---------------|----------|
| **LeetCode Explorer** | 100+ задач, топ-100 по темам | Начальное изучение тем |
| **Codeforces** | Полностью бесплатен | Скорость, pressure training |
| **neetcode.io** | Бесплатный roadmap + решения | Структурированный план |

#### Сравнение: почему Codeforces

| Критерий | Codeforces | Codewars | HackerRank |
|----------|------------|----------|------------|
| Тип задач | Олимпиадные, алгоритмические | Ката (языковые) | Учебные |
| Соответствие собеседованиям | Высокое | Низкое | Среднее |
| Community | Очень активная | Активная | Средняя |

**Codeforces лучше для собеседований, потому что:**
- Задачи A-C на Div2 соответствуют easy/medium на интервью
- Google, Meta, Amazon используют CF-подобные задачи
- Регулярные контесты = тренировка работы под давлением

**Codewars** — больше для изучения синтаксиса языка, не для алгоритмической подготовки.

### Ключевые принципы

1. Spaced repetition: возврат к пройденному через 1/3/7/14 дней
2. Active recall: минимум 70% времени — решение задач, 30% — изучение теории
3. Один язык: выбрать Java (есть опыт), все структуры реализовать на нём
4. Микрозадачи: не "изучить trees", а "решить задачу про инвертирование BST"

### Codeforces: темы и порядок

#### Level 1 — Базовые (1-3 недели)
| Тег CF | Что учит | Сложность |
|--------|----------|-----------|
| `implementation` | Внимательность, синтаксис | 800-1000 |
| `math` | GCD, LCM, модульная арифметика | 800-1000 |
| `strings` | Работа со строками | 800-1000 |
| `sorting` | Сортировки, компараторы | 800-1100 |

#### Level 2 — Core Algorithms (3-6 недель)
| Тег CF | Что учит | Сложность |
|--------|----------|-----------|
| `binary search` | Бинарный поиск | 1000-1200 |
| `two pointers` | Two pointers, sliding window | 1000-1200 |
| `prefix sum` | Префиксные суммы | 1000-1200 |
| `hashing` | Хеш-таблицы | 1000-1300 |
| `stack` | Стек, балансировка скобок | 1000-1200 |
| `queue` | Очереди, BFS | 1000-1200 |

#### Level 3 — Структуры данных (6-9 недель)
| Тег CF | Что учит | Сложность |
|--------|----------|-----------|
| `trees` | DFS/BFS на деревьях | 1200-1400 |
| `graphs` | Графы, Dijkstra | 1200-1400 |

#### Level 4 — Продвинутые (9+ недель)
| Тег CF | Что учит | Сложность |
|--------|----------|-----------|
| `dp` | Динамическое программирование | 1300-1600 |
| `bitmasks` | Битовые операции | 1000-1300 |

#### Порядок решения
```
1. math → strings → sorting (800-1000)
2. binary search → two pointers → hashing (1000-1200)
3. stack → queue → graphs (1000-1300)
4. dp (1300+)
```

#### Прогрессия сложности
- **800-1000** = Easy (10-15 мин)
- **1100-1300** = Medium (15-25 мин)
- **1400+** = Hard
