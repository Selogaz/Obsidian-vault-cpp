# Цикл работы с задачами

Полный GTD-цикл в этом хранилище: от захвата до закрытия.

## 1. Захват — `#task/inbox`

Любая мысль, задача, идея записывается с тегом `#task/inbox`. Писать можно **в любую заметку**: daily note, `home/fleetings.md`, `base/notes/` по теме, прямо в проект.

```markdown
- [ ] #task/inbox #category/health Купить витамин D
- [ ] #task/inbox #category/webdev Пофиксить баг с ховером
- [ ] #task/inbox Идея — посмотреть лекцию про векторы
```

Категория опциональна. Если не уверен — пиши без неё.

## 2. Автоматическая миграция

При запуске `today` skill собирает все `#task/inbox` из прошлых daily notes и переносит их в сегодняшний брифинг (`periodic/daily/YYYY-MM-DD.md`). Из исходных заметок они удаляются. Это автоматизация GTD-шага «process inbox».

## 3. Классификация

При обработке инбокса тег `inbox` заменяется на конкретный тип:

| Тип | Когда |
|-----|-------|
| `#task/one-off` | Сделать один раз, 5-30 мин |
| `#task/multistep` | Нужно несколько шагов (подзадачи) |
| `#task/next_action` | Сделать прямо сейчас, приоритет |
| `#task/waiting_for` | Жду ответа / внешнего действия |
| `#task/regular` | Повторяющаяся задача |
| `#task/idea` | Идея, а не задача |
| `#task/someday` | Когда-нибудь потом |
| `#task/reference` | Почитать / посмотреть |

```markdown
- [ ] #task/inbox Купить витамин D
→ - [ ] #task/one-off #category/health Купить витамин D

- [ ] #task/inbox Сделать оффлайн AI-приложение для языков
→ - [ ] #task/idea #category/english #category/webdev Сделать оффлайн AI-приложение для языков
```

Категория (`#category/<name>`) добавляется на этом этапе. Список существующих категорий — `base/categories/`.

## 4. Дашборд — `periodic/tasks.md`

Главная панель задач. Содержит ссылки на все типы:

- `[[inbox]]` — необработанные `#task/inbox`
- `[[today]]` — задачи на сегодня (`#task/next_action`)
- `[[upcoming]]` — запланированное
- `[[calendar]]` — календарь с дедлайнами
- `[[priority]]` — приоритеты
- `[[idea]]` —идеи
- `[[next_action]]` — следующее действие
- `[[one-off]]` — простые задачи
- `[[multistep]]` — проекты с подзадачами
- `[[someday]]` — «когда-нибудь»
- `[[reference]]` — ресурсы на потом
- `[[waiting_for]]` — ожидающие внешнего действия

## 5. Fleetings — `home/fleetings.md`

Стартовая точка для просмотра накопленного. Вкладки: осиротевшие заметки, недавние изменения, все `#task/inbox` через Dataview, и задачные заметки (`base/tasks/`).

## 6. Заметки-задачи — `base/tasks/`

Для `#task/multistep` (проектов с подзадачами) можно создать отдельную заметку в `base/tasks/` с подробным описанием, чеклистом, ссылками.

## Типичный день

1. Открываешь daily note → видишь Briefing со старыми `#task/inbox` (уже перенесены AI)
2. Обрабатываешь inbox → меняешь теги на `one-off` / `idea` / `waiting_for` / и т.д.
3. Работаешь → создаёшь новые `#task/inbox` в процессе
4. На следующий день AI снова собирает оставшиеся `#task/inbox` в брифинг

`inbox` — временное состояние «ещё не разобрал». Как только задача осмыслена, она должна получить конкретный тип. Даже `#task/someday` лучше, чем вечно висеть в inbox.

Дальше — [[конвейер знаний от источника до иерархии]]: задачи превращаются в источники, источники — в заметки, заметки — в иерархии.
