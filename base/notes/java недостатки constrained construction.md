**недостатки constrained construction**
—
- **Ограничивает дизайн классов** - вынуждает создавать конструкторы под конкретный способ создания ([[java Reflection]], а не под удобство использования
- **Нарушает инкапсуляцию** - чтобы reflection мог создать объект, часто нужен пустой конструктор, что вынуждает делать поля изменяемыми или использовать сеттеры
- **Сложность с immutability** - нельзя сделать объект полностью неизменяемым, если нужен пустой конструктор для Reflection
- **Жесткая связь** - код создания через reflection знает о внутреннем устройстве классов
Вместо него лучше использовать DI-контейнер (Spring, Dagger) или фабрики
