**ADR-003: Слой данных — Тип → Сервис → Хук**
—
**Контекст.** Фронт работает с внешним backend (`https://test.dndmaxwell.online`) без локального API. Часть эндпоинтов готова, часть — нет (лендинг).

**Решение.** Единообразный слой данных для всех блоков:

```
src/types/*.ts          Определяет интерфейс данных
    ↓
src/services/*.ts       Функция получения данных (fetch / заглушка)
    ↓
src/hooks/*.ts          Состояния: data, loading, error, refetch
    ↓
Компонент               Использует хук
```

- Для готового API — сервис делает `fetch()` через прокси
- Для заглушек — сервис возвращает захардкоженный JSON, сохраняя сигнатуру

**Прокси-слой** (`src/proxy.ts`): Next.js middleware, удаляет `Origin`, реврайт на backend для `/api/auth/*`, `/api/me`, `/api/character`, `/api/character-builder/*`.

**Server-side route** (для эндпоинтов, проверяющих Origin): `src/app/api/game/start/route.ts`.

**Последствия.**
- + замена заглушки на реальный API — изменить одно тело сервиса, не меняя типы и хуки
- + единый паттерн для всех данных — легко читать и расширять
- + retry-прослойка (`_retry.ts`) применяется централизованно
