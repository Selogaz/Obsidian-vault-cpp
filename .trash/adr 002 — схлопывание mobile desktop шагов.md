**ADR-002: Схлопывание mobile/desktop шагов визарда**
—
**Контекст.** Каждый шаг визарда персонажа изначально существовал в двух файлах: `*Step.tsx` (desktop) и `*StepMobile.tsx` (mobile). Это давало 18 файлов на 9 шагов, ~5.5k строк.

**Проблема.**
- диффы между парами desktop/mobile расходились (фикс в одном, забыли в другом)
- любая новая фича оплачивалась дважды
- page.tsx держал два параллельных switch-блока с одинаковыми пропсами

**Рассматривались варианты:**
1. **Headless-хук + два View.** Общая логика в хуке, тонкие вьюхи. Минус: всё ещё два файла, можно опять разъехаться.
2. **Один компонент с responsive-блоками** (`hidden md:flex` / `md:hidden`). Логика одна, вёрстка разделена внутри файла.

**Решение.** Вариант 2: один файл на шаг, desktop-вёрстка в `hidden md:flex`, mobile — в `md:hidden`. Tailwind-кастомные варианты `dol:` (desktop-or-landscape), `nlp:` (narrow landscape phone), `tab:` (планшеты) для точной настройки.

**Исключение.** Если файл превышает ~500 строк — допускается выделить mobile в отдельный `*StepMobile.tsx`.

**Последствия.**
- + единый источник правды для логики шага
- + нельзя забыть перенести фикс на mobile
- + page.tsx сжимается до одной switch-таблицы
- файлы крупнее (десктоп + mobile в одном)
