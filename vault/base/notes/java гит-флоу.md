**гит-флоу**
—
Гит-флоу — это конкретный паттерн поведения при работе над полученной задачей или багом, касаемых разработки. У вас есть девелоп ветка с актуальными фичами в приложении, вы получили новую задачу для разработки нового функционала, поэтому от девелопа надо отбренчиться и создать новую ветку таким образом - номер задачи из тикета/feature/Название задачи из тикета. В этой ветке необходимо будет сделать всю работу над фичей и запушить ее в ваш репозиторий. Это описание самого простого гитфлоу, в каждой команде он может отличаться неймингом веток, основной веткой (девелом, мейн, мастер), а также количеством комитов и их длинной
