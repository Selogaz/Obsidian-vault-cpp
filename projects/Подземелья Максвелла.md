## ТЗ

Задача — не просто натянуть экраны, а собрать нормальный фронт приложения на Next.js + React + TypeScript + Tailwind.

Верстка должна быть компонентной: все повторяющиеся элементы из макетов выносить в переиспользуемые React-компоненты.
Не дублировать одинаковую разметку по страницам, а собирать проект через layouts, pages, sections, UI-components.

Фронт нужно сразу делать так, чтобы потом в него без переделок подключались:

API
авторизация
состояния
websocket-логика
остальная бизнес-логика

По адаптиву:

основной режим — desktop-first
обязательно сразу делать mobile-адаптацию
сложные блоки на узких экранах должны нормально перестраиваться, а не просто сжиматься

В верстке нужно учитывать состояния интерфейса:

hover
focus
active
disabled
error
loading
empty states
success states

Важно:

не делать набор отдельных html-страниц
не складывать все в один файл
не плодить уникальную верстку там, где должен быть переиспользуемый компонент
не делать одноразовую структуру, которую потом придется переписывать

Итог: нужен не набор сверстанных макетов, а нормально собранный фронт приложения, готовый к дальнейшему подключению backend API и логики.

## UI-компоненты (базовые)

Первоочередные переиспользуемые компоненты:

- [ ] `Button` — primary, secondary, ghost, disabled states
- [ ] `Input` — text, email, password, с error state
- [ ] `Card` — контейнер с hover/focus
- [ ] `Modal` — для подтверждений
- [ ] `Badge` — статусы, теги
- [ ] `Avatar` — для профиля
- [ ] `Select` — выпадающие списки
- [ ] `Tooltip` — подсказки

## Состояния интерфейса

Для каждого интерактивного элемента предусмотреть:
- `:hover` — подсветка
- `:focus` — доступность с клавиатуры
- `:active` — нажатие
- `:disabled` — неактивное состояние
- `error` — валидационные ошибки
- `loading` — лоадеры (spinner/skeleton)
- `empty` — пустые состояния
- `success` — успешные операции

## 🟥 Секция "Играть" (Личный кабинет)

Страница доступна из боковой навигации ЛК. Три основных действия:

### Опции страницы

| Опция | Описание |
|-------|----------|
| **Создать игру** | Создание новой кампании/сессии |
| **Продолжить игру** | Возврат к активной игре |
| **Присоединиться к игре** | Подключение к существующей кампании по коду/ссылке |

### UI-компоненты для секции

- [ ] `GameActionCard` — карточка с иконкой и CTA для каждого действия
- [ ] `GameCodeInput` — поле ввода кода присоединения к игре
- [ ] `ActiveGameList` — список активных игр для "Продолжить"
- [ ] `CreateGameModal` — модалка создания новой игры (настройки кампании)

### Состояния интерфейса

- Пустое состояние (нет активных игр) — приглашение создать или присоединиться
- Загрузка — спиннер при получении списка игр
- Ошибка — при неудачном подключении или создании
- Успех — подтверждение присоединения/создания
