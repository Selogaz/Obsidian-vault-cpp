## ТЗ

Задача — не просто натянуть экраны, а собрать нормальный фронт приложения на Next.js + React + TypeScript + Tailwind.

Верстка должна быть компонентной: все повторяющиеся элементы из макетов выносить в переиспользуемые React-компоненты.
Не дублировать одинаковую разметку по страницам, а собирать проект через layouts, pages, sections, UI-components.

Фронт нужно сразу делать так, чтобы потом в него без переделок подключались:

API
авторизация
состояния
websocket-логика
остальная бизнес-логика

По адаптиву:

основной режим — desktop-first
обязательно сразу делать mobile-адаптацию
сложные блоки на узких экранах должны нормально перестраиваться, а не просто сжиматься

В верстке нужно учитывать состояния интерфейса:

hover
focus
active
disabled
error
loading
empty states
success states

Важно:

не делать набор отдельных html-страниц
не складывать все в один файл
не плодить уникальную верстку там, где должен быть переиспользуемый компонент
не делать одноразовую структуру, которую потом придется переписывать

Итог: нужен не набор сверстанных макетов, а нормально собранный фронт приложения, готовый к дальнейшему подключению backend API и логики.
