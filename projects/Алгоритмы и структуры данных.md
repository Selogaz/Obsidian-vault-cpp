### Понимание условия задачи

- Перечитать условие. Не забыть примечания и конец
- Выписать дано и найти
	- Размер массива, ограничения на значения, тип данных
- Прорешать примеры вручную
	- пройти примеры из условия шаг за шагом
	- придумать свой маленький пример
- Вопросы себе:
	- Что именно нужно оптимизировать?(Время? Память? Количество операций?)
	- Есть ли ограничения на знаки чисел?(отрицательные могут быть? Нули?)
	- Могут ли элементы повторяться?
	- Что будет, если массив пустой или состоит из одного элемента?

### Анализ ограничений и выбор сложности

Посмотреть на число N и время выполнения(обычно 1 секунда). Это подскажет какой алгоритм нужен.

выбор [[java алгоритмическая сложность|алгоритмической сложности]]:
- N <= 20 - можно О(2^N) (рекурсия, битовые маски)
- N <= 100 -> Можно О(N^3) (тройной цикл)
- N <= 2000 -> О(N^2) (динамическое программирование, парные циклы)
- N <= 10^5...10^6 О(N) или О(N log N) (сортировка, жадность, два указателя, бинпоиск)
- N >= 10^9 О(log N) или О(1) (математическая формула, бинпоиск по ответу)
- Вопрос себе: уложится ли мой перебор в 1 секунду?(примерно 10^8 операций в секунду)

### Генерация идеи

- Решить задачу в лоб (brute force). **Как решить, если бы ограничений не было?**
- Поиск закономерностей
	- Если отсортировать массив, станет ли проще?
	- Можно ли разбить задачу на подзадачи?
	- Зависит ли ответ только от последнего элемента?(Признак ДП)
	- Можно ли брать локально лучшее решение?(Признак жадности)
- Вопросы себе
	- Что не меняется в процессе?(Инварианты)
	- Могу ли я зафиксировать один параметр и перебрать другой?
	- Похожа ли эта задача на что-то известное?(Поиск пути, сумма подотрезка, связность)

### Проектирование алгоритма

- Псевдокод. Основные шаги словами
- Структуры данных. Что подойдет?
	- Быстрый поиск? -> Set, Map, [[HashMap]]
	- Нужен порядок? -> [[PriorityQueue]]
	- Нужны диапазоны? -> Дерево отрезков/префиксные суммы
- Вопросы себе
	- Хватит ли памяти?(Не создаю ли я массив 10^9?)
	- Не переполнится ли тип данных? (Нужен ли long вместо инт?)

### Реализация

- Шаблоны. Используй свои заготовки
- Модульность. Сложную задачу разбить на функции
- Вопросы себе
	- Правильно ли я инициализировал переменные?(флаги, суммы)
	- Нет ли выхода за границы массива?

### Тестирование и Corner Cases

Проверка перед отправкой:
- Минимальные значения: N = 1, N = 0
- Максимальные значения
- Одинаковые элементы
- Уже отсортировано
- Отрицательные числа/нули
- Что сломает код, если я подставлю экстремальное значение?

DBMB and the Array
![[Алгоритмы и структуры данных 2026-03-09.png]]
—
первое число - n, второе - s, третье - x
n - количество чисел во второй строке
s - число, которой должна быть равна сумма чисел во второй строке. Если этого(числа s) можно достичь, прибавляя x к одному из чисел во второй строке, то выводить YES. Иначе NO
Также s должно быть меньше суммы чисел.
```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        int cycleCounter = input.nextInt();
        input.nextLine();//для перехода на новую строку, ведь nextInt() не считывает символ конца строки
        while (cycleCounter > 0) {
            int sum = 0;
            int n = input.nextInt();
            int s = input.nextInt();
            int x = input.nextInt();
            input.nextLine();

            int[] arr = new int[n];
            String[] secondLine = input.nextLine().split(" ");
            for (int i = 0; i < n; i++) {
                arr[i] = Integer.parseInt(secondLine[i]);
                sum += arr[i];
            }

            if (sum == s) {
                System.out.println("YES");
            } else if (((s - sum) % x == 0) && (sum < s)) {
                System.out.println("YES");
            } else {
                System.out.println("NO");
            }
            cycleCounter--;
        }
    }
}
```

## Shifted MEX
![[Алгоритмы и структуры данных 2026-03-10.png]]
- Первая строка - t - число тестов (от 1 до 1000)
- Вторая строка - n - длина массива а (до 3000)
- Третья строка - элементы массива a через пробел
- К элементам массива можно прибавить или отнять одно число (x)
- Выход программы: наименьшее не отрицательное число, которое не представлено в массиве после применения операции с числом х. Если кратко, то наименьшее число, которого нет в массиве после суеты с х. Х выбирается таким образом, чтобы получилась максимально длинная цепочка, начиная с 0. Например 0, 1,2,...L-1. В данном случае MEX будет L.
—
- Как вычислять длину цепочки, где идут числа по возрастанию?
	- если следующий элемент больше текущего, то длина цепочки++
- Как её хранить?
	- в отдельной переменной
- Как обновлять?
	- Когда текущая длина больше максимальной
- Как понять с какого индекса началась длинная цепочка?
	- От индекса элемента, где замечена максимальная цепочка, вычесть длину максимальной цепочки
- Проблема: алгоритм не считает цепочку в последнем элементе массива. Как решить?
	- Сравнивать не следующий элемент с текущим, а **текущий с предыдущим**, начиная со второго. МЕХ первого всегда будет равен 1.

Псевдокод:
- Отсортировать массив
- Найти Х
- Вычесть Х из массива
- Определить МЕХ

мой старый код
```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        int cycleCounter = input.nextInt();
        input.nextLine();
        while (cycleCounter > 0) {
            int n = input.nextInt();
            input.nextLine();

            Set<Integer> sortedSet = new TreeSet<>();
            String[] secondLine = input.nextLine().split(" ");
            for (int i = 0; i < n; i++) {
                sortedSet.add(Integer.parseInt(secondLine[i]));
            }
            System.out.println(sortedSet);

            boolean firstItemFlag = false;
            int firstItem = 0;
            int expected = 0;
            Set<Integer> shiftedSet = new TreeSet<>();
            for (Integer elem : sortedSet) {
                if (!firstItemFlag) {
                    firstItem = elem;
                    firstItemFlag = true;
                }
                firstItem = -firstItem;
                shiftedSet.add(elem + firstItem);
                if (elem == expected) {
                    expected++;
                }
            }
            System.out.println(shiftedSet);
            System.out.println("MEX: " + expected);

            int newExpected = 0;
            for (Integer elem : shiftedSet) {
                if (elem == newExpected) {
                    newExpected++;
                }
            }
            System.out.println("Shifted MEX: " + newExpected);
            cycleCounter--;
        }
    }
}
```

## Bit++
The classic programming language of Bitland is Bit++. This language is so peculiar and complicated.
The language is that peculiar as it has exactly one variable, called x. Also, there are two operations:

    Operation ++ increases the value of variable x by 1.
    Operation -- decreases the value of variable x by 1. 
    
A statement in language Bit++ is a sequence, consisting of exactly one operation and one variable x. The statement is written without spaces, that is, it can only contain characters "+", "-", "X". Executing a statement means applying the operation it contains.
A programme in Bit++ is a sequence of statements, each of them needs to be executed. Executing a programme means executing all the statements it contains.
You're given a programme in language Bit++. The initial value of x is 0. Execute the programme and find its final value (the value of the variable when this programme is executed).
Input
The first line contains a single integer n (1 ≤ n ≤ 150) — the number of statements in the programme.
Next n lines contain a statement each. Each statement contains exactly one operation (++ or --) and exactly one variable x (denoted as letter «X»). Thus, there are no empty statements. The operation and the variable can be written in any order.
Output
Print a single integer — the final value of x.
```
Examples
Input

1
++X

Output

1

Input

2
X++
--X

Output

0
```

import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        int cycleCounter = input.nextInt();
        input.nextLine();
        while (cycleCounter > 0) {
            int initialValue = 0;

//            String[] secondLine = input.nextLine().split(" ");
            String secondLine = input.nextLine();
            switch(secondLine) {
                case "++X":
                    ++initialValue;
                    break;
                case "X++":
                    initialValue++;
                    break;
                case "--X":
                    --initialValue;
                    break;
                case "X--":
                    initialValue--;
                    break;
                default:
                    break;

            }
            System.out.println(initialValue);
            cycleCounter--;
        }
    }
}
