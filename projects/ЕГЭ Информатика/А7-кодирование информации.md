# Кодирование изображений

## 🗂️ КАТЕГОРИЯ 1: Кодирование графической информации (изображения)

### 📐 Основные формулы:

```zsh
1) N = 2^i          → количество цветов (N) через глубину цвета (i)
2) i = log₂N        → глубина цвета через количество цветов
3) V = K × i        → объём изображения в битах
   где:
   • K = ширина × высота (общее число пикселей)
   • i — бит на пиксель (глубина цвета)
   • V — объём в битах
```

### 🔍 Типы задач:

#### Тип 1.1: Найти количество цветов

> Дано: размер изображения, объём памяти. Найти: максимальное число цветов.

**Алгоритм:**

1. Найти общее число пикселей: `K = ширина × высота`
2. Перевести объём памяти в биты
3. Найти бит на пиксель: `i = V / K` (округлить **вниз** до целого!)
4. Найти количество цветов: `N = 2^i`

#### Тип 1.2: Найти объём памяти

> Дано: размер изображения, количество цветов. Найти: объём файла.

**Алгоритм:**

1. Найти глубину цвета: `i = log₂N` (округлить **вверх**!) $i =\frac{V}{K}$
2. Найти число пикселей: `K = ширина × высота`
3. Объём в битах: `V = K × i`
4. Перевести в нужные единицы (байты, Кбайт...)

#### Тип 1.3: Задачи со сжатием/архивацией

> Дано: размер сжатого файла, процент сжатия. Найти: количество цветов.

**Алгоритм:**

1. Найти объём исходного файла: `V_исх = V_сжат / (1 - процент_сжатия)`
2. Далее как в Типе 1.1

⚠️ **Важно:** если в условии сказано «файл больше сжатого на 35%», значит сжатый = 65% от исходного!

### Примеры задач

Для хранения произвольного растрового изображения размером 1536×2048 пикселей отведено не более 6 Мбайт памяти без учёта размера заголовка файла. Для кодирования цвета каждого пикселя используется одинаковое количество бит, коды пикселей записываются в файл один за другим без промежутков. Какое максимальное количество цветов можно использовать в изображении?
—
Максимальный объем изображения равен $6 Мбайт = 6⋅ 2^{23}  бит$
Количество цветов ищем по формуле $2^i$, где i - объем памяти, необходимый для хранения цвета одного пикселя. 
Чтобы найти i, разделим максимальный объем файла на количество пикселей в изображении.
Получаем дробь:
$i=\frac{6 ⋅ 2^{23}}{1536 ⋅ 2048}=16 бит$,
$2^{16}=65536$ цветов
1. Пиксели = ширина × высота
2. Доступно бит = Мбайт × 1024 × 1024 × 8
3. n ≤ Доступно бит / Пиксели (округляем вниз до целого)
4. Количество цветов = 2^n

Сколько секунд потребуется модему, передающему сообщения со скоростью 14400 бит/с, чтобы передать цветное растровое изображение размером 640×480 пикселей, при условии, что цвет каждого пикселя кодируется 24 битами?
—
Чтобы найти *объем изображения*, умножим *количество пикселей* на *объем памяти*, необходимый для хранения цвета одного пикселя. Чтобы найти *время передачи сообщения*, разделим *объем файла* на *скорость передачи*.
Время передачи файла:
$t=\frac{V}{\upsilon}$, где V - объем изображения, $\upsilon$ - скорость передачи
Объем изображения:
$V=K\cdot i=640\cdot 480\cdot 24$
Получаем дробь:
$i = \frac{640~\cdot~480~\cdot~24}{14400} = 512~секунд.$

# Кодирование звука

## 🗂️ КАТЕГОРИЯ 2: Кодирование звуковой информации

### 📐 Основная формула:

```zsh
V = ν × i × k × t

где:
• V — объём файла в битах
• ν (ню) — частота дискретизации в Гц (отсчётов в секунду)
• i — глубина кодирования в битах (бит на отсчёт)
• k — количество каналов (моно=1, стерео=2, квадро=4)
• t — время записи в секундах
```

zftsh.online

### 🔍 Типы задач:

#### Тип 2.1: Найти объём звукового файла

> Дано: ν, i, k, t. Найти: V.

**Решение:** Просто подставить в формулу `V = ν × i × k × t`

#### Тип 2.2: Найти время записи

> Дано: объём файла, ν, i, k. Найти: t.

**Решение:** `t = V / (ν × i × k)`

#### Тип 2.3: Найти глубину кодирования или частоту

> Дано: объём, время, остальные параметры. Найти: недостающий параметр.

**Решение:** Выразить неизвестное из основной формулы.

⚠️ **Важно:**

- 1 кГц = 1000 Гц (в ЕГЭ обычно используют именно 1000, а не 1024!)
- Переводите минуты в секунды: `t(сек) = t(мин) × 60`

# Передача данных по каналу связи

## КАТЕГОРИЯ 3: Передача данных по каналу связи

### 📐 Основная формула:

```zsh
V = q × t

где:
• V — объём переданных данных в битах
• q — скорость передачи в бит/с
• t — время передачи в секундах
```

ege-study.ru

### 🔍 Типы задач:

#### Тип 3.1: Найти время передачи

> Дано: объём файла, скорость канала. Найти: время.

**Решение:** `t = V / q`

#### Тип 3.2: Найти скорость канала

> Дано: объём, время. Найти: скорость.

**Решение:** `q = V / t`

#### Тип 3.3: Комплексные задачи (изображение/звук + передача)

> Дано: параметры изображения/звука + скорость канала. Найти: время передачи.

**Алгоритм:**

1. Сначала найти объём данных по формулам Категории 1 или 2
2. Затем найти время: `t = V / q`

#### Тип 3.4: Максимальное количество файлов в пакете

> Дано: время на передачу пакета, скорость, параметры одного файла. Найти: сколько файлов поместится.

**Алгоритм:**

1. Найти объём одного файла: `V₁`
2. Составить неравенство: `(V₁ × X) / q ≤ t_макс`
3. Решить относительно X, округлить **вниз** до целого

# Кодирование текста(реже)

```zsh
V = n × i

где:
• n — количество символов
• i — бит на символ (зависит от кодировки)
```
### 📊 Типичные кодировки:

|Кодировка|Бит на символ|Примечание|
|---|---|---|
|ASCII|7-8 бит|Только латиница|
|ANSI/Windows-1251|8 бит (1 байт)|Кириллица|
|UTF-8|1-4 байта|Переменная длина|
|UTF-16|2 или 4 байта|Фиксированная для БМП|
# Частые ошибки (избегайте!)

|Ошибка|Как правильно|
|---|---|
|Забыли перевести Мбайт в биты|Всегда переводите в биты перед расчётами!|
|Округлили не в ту сторону|`i` (бит на пиксель) — округлять **вверх**; `X` (количество файлов) — **вниз**|
|Перепутали k для звука|Моно=1, Стерео=2, Квадро=4|
|Использовали 1024 вместо 1000 для кГц|В звуковых задачах 1 кГц = 1000 Гц|
|Не учли, что палитра 7026 цветов требует 13 бит|`2¹²=4096 < 7026`, значит нужно 13 бит!<br><br>ege-study.ru|
